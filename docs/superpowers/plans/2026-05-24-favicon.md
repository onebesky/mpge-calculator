# Favicon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a bolt + gas-drop favicon (SVG primary, 32×32 PNG fallback) to the True MPG site.

**Architecture:** Two static asset files (`favicon.svg`, `favicon.png`) added to the project root. Two `<link>` tags wired into `index.html`. PNG is rasterized from the SVG using a one-off Node.js + sharp script run in a temp directory so the project stays dependency-free.

**Tech Stack:** SVG, Node.js 24 (available), sharp (installed temporarily in /tmp)

---

### Task 1: Create favicon.svg

**Files:**
- Create: `favicon.svg`

- [ ] **Step 1: Write the SVG file**

Create `favicon.svg` in the project root with this exact content:

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 72 72">
  <polygon points="28,8 16,38 27,38 22,64 42,30 31,30 40,8" fill="#5fd08a"/>
  <path d="M52 22 C52 22 44 32 44 38 C44 44 47.5 48 52 48 C56.5 48 60 44 60 38 C60 32 52 22 52 22Z" fill="#e8843c"/>
</svg>
```

The bolt uses EV green (`#5fd08a`) and the gas drop uses gas orange (`#e8843c`), matching the app's CSS variables. No background — transparent.

- [ ] **Step 2: Verify the SVG renders correctly**

Open `favicon.svg` directly in a browser tab. You should see a green lightning bolt on the left and an orange teardrop on the right against a transparent/white background. Both shapes should be clearly visible.

### Task 2: Generate favicon.png

**Files:**
- Create: `favicon.png`

- [ ] **Step 1: Set up a temp sharp environment**

Run in terminal (not in the project directory):

```bash
mkdir -p /tmp/favicon-gen
cd /tmp/favicon-gen
npm init -y --quiet
npm install sharp --save-quiet
```

Expected output ends with: `added N packages`

- [ ] **Step 2: Run the conversion**

Still in `/tmp/favicon-gen`, run:

```bash
node -e "
const sharp = require('sharp');
const fs = require('fs');
const svg = fs.readFileSync('/Users/ondrejnebesky/workspace/mpge-calculator/favicon.svg');
sharp(svg)
  .resize(32, 32)
  .png()
  .toFile('/Users/ondrejnebesky/workspace/mpge-calculator/favicon.png', (err) => {
    if (err) throw err;
    console.log('favicon.png created');
  });
"
```

Expected output: `favicon.png created`

- [ ] **Step 3: Verify the PNG**

Back in the project directory, check the file exists and is a reasonable size:

```bash
ls -lh /Users/ondrejnebesky/workspace/mpge-calculator/favicon.png
```

Expected: file exists, size between 200B and 5KB. Open it in Preview to confirm you see the bolt + drop at 32×32.

- [ ] **Step 4: Clean up the temp directory**

```bash
rm -rf /tmp/favicon-gen
```

### Task 3: Wire up favicon in index.html

**Files:**
- Modify: `index.html` — add two `<link>` tags inside `<head>`

- [ ] **Step 1: Add the link tags**

In `index.html`, locate the existing `<meta name="viewport">` tag (line 6). Add the two favicon links immediately after it:

```html
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <link rel="icon" type="image/svg+xml" href="favicon.svg">
  <link rel="icon" type="image/png" href="favicon.png">
  <title>True MPG — what your MPGe actually costs</title>
```

The SVG link must come first — browsers prefer the first matching type, so modern browsers pick SVG and older browsers fall back to PNG.

- [ ] **Step 2: Verify in the browser**

Open `index.html` in a browser (or reload if already open). Check the browser tab — you should see the bolt + drop icon appear. If the tab shows a blank page icon, hard-refresh with Cmd+Shift+R.

- [ ] **Step 3: Test on a light browser theme**

If your browser supports theme switching, toggle to a light theme and confirm the transparent favicon still reads clearly (green bolt and orange drop should be visible on the white/grey tab bar).
